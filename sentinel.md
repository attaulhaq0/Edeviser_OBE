## 2025-04-21 - [CRITICAL] Missing Authentication in Edge Functions
**Vulnerability:** Several Edge Functions in `supabase/functions/` (such as `bulk-data-import`, `import-competency-csv`, `generate-transcript`) initialize the Supabase client directly with `SUPABASE_SERVICE_ROLE_KEY` without first authenticating the caller via the `Authorization` header. This allows unauthenticated attackers to execute arbitrary actions such as reading, inserting, or modifying data in the database with elevated privileges.
**Learning:** The service role key bypasses Row Level Security (RLS). When creating an Edge Function that relies on the service role key to perform operations, it is critical to implement a manual authentication and authorization check first to ensure only allowed users can invoke the function.
**Prevention:** Always validate the caller's JWT by extracting the `Authorization` header, initializing a user-specific client using `SUPABASE_ANON_KEY` and the authorization header (`{ global: { headers: { Authorization: authHeader } } }`), and verifying the user with `userClient.auth.getUser()` before processing the request or using `SUPABASE_SERVICE_ROLE_KEY`.

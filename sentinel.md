## 2025-02-14 - Unauthorized Access to Bulk Data Import Edge Function
**Vulnerability:** The `bulk-data-import` Supabase Edge Function lacked authentication checks, relying entirely on the `SUPABASE_SERVICE_ROLE_KEY` to access the database. This allowed any unauthenticated user to invoke the endpoint, bypassing Row Level Security (RLS), and potentially spoofing the `performed_by` field to attribute malicious data imports to other users.
**Learning:** Edge Functions that use the Service Role key bypass RLS completely. If they are publicly accessible (no JWT validation), they create severe data integrity and security risks. Additionally, relying on client-provided data for critical audit fields (like `performed_by`) without server-side validation is a vector for spoofing.
**Prevention:** Always validate the caller's JWT by extracting the `Authorization` header and initializing a user-specific client using `SUPABASE_ANON_KEY`. Verify the user via `userClient.auth.getUser()`. Extract the user's ID securely from the validated session rather than trusting client payloads.

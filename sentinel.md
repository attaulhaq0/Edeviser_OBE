## 2024-05-15 - [CRITICAL] Missing Authentication in Edge Functions
**Vulnerability:** Several edge functions, including `bulk-data-import` and `generate-transcript`, completely lacked authentication checks. They used the `SUPABASE_SERVICE_ROLE_KEY` to interact with the database without verifying the caller's identity via the `Authorization` header. This allowed unauthenticated, unauthorized users to perform administrative actions and access sensitive data.
**Learning:** Edge functions are publicly accessible endpoints. Using the service role key grants the function elevated privileges, but it does not authenticate the user *calling* the function. Developers mistakenly assumed that calling the function from an authenticated client application automatically protected the function, forgetting to implement explicit authentication inside the edge function itself.
**Prevention:** Always authenticate the caller inside an edge function by extracting the `Authorization` header, creating a Supabase client using the `SUPABASE_ANON_KEY` and the authorization header, and calling `auth.getUser()`. Only use the `SUPABASE_SERVICE_ROLE_KEY` for subsequent database operations *after* verifying the user's identity and permissions (e.g., role).

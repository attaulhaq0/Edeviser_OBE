## DEFER rationale

- **unused_index (~110, INFO):** The DB is at near-zero real traffic (test/seed data only). Postgres marks an index "unused" only because no query has hit it yet. These indexes were deliberately created to cover FK joins and common filters (institution_id, course_id, semester, search columns) that WILL be exercised once real students/teachers use the platform. Dropping them now risks removing indexes that become hot under real load and would require re-creation under production traffic. Deferred to a post-launch review once `pg_stat_user_indexes` reflects real usage. NOT a gate.
- **multiple_permissive_policies (hundreds, WARN):** Each finding is two or more permissive policies for the same (role, action) — by design, the platform layers role-scoped policies (admin/teacher/student/parent each get their own permissive SELECT). Consolidating them into single combined policies is a large RLS redesign that changes the allowed/denied matrix and risks both over- and under-exposure. This is a correctness-sensitive refactor, not a drift fix, and must be designed + per-role probe-tested separately. Deferred. NOT a gate.
